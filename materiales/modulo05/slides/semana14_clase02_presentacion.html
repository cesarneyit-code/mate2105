La pregunta matemática

¿Qué propiedad verifica un hash criptográfico y qué queda fuera de esa verificación?

Expresión central

\[ h=\operatorname{SHA256}(\text{bytes del archivo}). \]

Hashes iguales implican identidad de bytes con probabilidad práctica muy alta. No implican datos válidos, código correcto ni inferencia adecuada.

Primer paso de la derivación

El manifiesto fija una función de reconstrucción: entradas + código + entorno + parámetros deben producir artefactos y métricas equivalentes dentro de tolerancia.

Segundo paso de la derivación

Si cambia un byte del dataset o modelo, cambia el hash. Para resultados numéricos puede requerirse tolerancia por plataforma, pero el cambio debe registrarse.

Qué debe coincidir

Métricas se almacenan por partición y nombre; un escalar score sin contexto no permite saber qué se midió ni dónde.

Una igualdad con dimensiones incompatibles no puede arreglarse con código.

Lectura del resultado

Un commit identifica código versionado; un entorno identifica dependencias; una semilla reconstruye elecciones pseudoaleatorias cuando la biblioteca es determinista.

Implementación mínima

def sha256_file(path):
    digest = hashlib.sha256()
    with open(path, "rb") as handle:
        for block in iter(lambda: handle.read(65536), b""):
            digest.update(block)
    return digest.hexdigest()

manifest["artifacts"][str(model_path)] = sha256_file(model_path)

El hash se calcula después de cerrar el archivo y sobre la versión que realmente se entregará.

Comprobación numérica

Valide JSON, recalcule hashes y ejecute el pipeline en una carpeta sin rutas absolutas. Compare métricas con tolerancia declarada.

Primero verificamos una propiedad matemática; después interpretamos la métrica.

Primer modo de falla

Rutas locales absolutas hacen que la corrida dependa del computador del autor.

Señal observable aparece /Users/... en código o manifest

Segundo modo de falla

Guardar solo requirements.txt sin versiones puede reconstruir un entorno distinto meses después.

Señal observable dependencias no tienen versión ni archivo de lock

Segunda pausa

El hash del modelo coincide, pero la tabla de métricas no. Diseñe una secuencia de auditoría.

Tiempo: 7 minutos. Proponga una comprobación que pueda ejecutarse.

Discusión de la segunda pausa

Verificar hash de datos, commit, parámetros y partición; cargar el modelo y recomputar métricas; comparar definición y orden de etiquetas. El hash solo identifica el modelo.

Qué sí permite afirmar

Un manifiesto con entorno, parámetros, métricas y artefactos conecta una corrida con archivos verificables y una ruta de reconstrucción.

Qué no permite afirmar

Reproducibilidad computacional no demuestra validez externa, ausencia de sesgo ni corrección de la pregunta.
