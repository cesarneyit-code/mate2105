Propósito del módulo

Este libro acompaña el primer bloque matemático fuerte del curso. La meta no es memorizar una lista de modelos, sino aprender un patrón de trabajo que se repetirá en ciencia de datos aplicada:

pregunta -> datos -> representación -> modelo -> pérdida
         -> algoritmo -> métrica -> diagnóstico -> decisión

En cinco semanas construiremos una cadena completa:

  1. regresión lineal como modelo matricial;
  2. mínimos cuadrados como problema de optimización;
  3. descenso del gradiente como algoritmo general;
  4. validación y regularización como control de complejidad;
  5. regresión logística como puente hacia clasificación.

Tesis del módulo. Un modelo no es una respuesta por sí solo. Un modelo es una hipótesis cuantitativa que debe venir con una pérdida, una métrica, una comparación contra baseline, una validación fuera de entrenamiento y una interpretación de sus limitaciones.

Cómo usar este libro

Cada capítulo tiene cuatro niveles de lectura.

Nivel Qué hacer Evidencia de que lo lograste
Conceptual Leer definiciones, notas e interpretaciones Puedes explicar el objeto sin código
Matemático Seguir derivaciones y proposiciones Puedes reproducir los pasos centrales
Computacional Leer e implementar los patrones NumPy Puedes escribir funciones pequeñas sin copiar
Aplicado Resolver ejercicios y conectar con laboratorios Puedes justificar una decisión técnica

No intentes estudiar el módulo solo ejecutando notebooks. El notebook muestra el cálculo en acción; el libro explica por qué ese cálculo tiene sentido.

Prerrequisitos mínimos

Antes de empezar, debes estar cómodo con:

  • vectores, matrices, transpuesta y producto matricial;
  • derivadas parciales básicas;
  • interpretación de una función de pérdida;
  • NumPy: arreglos, broadcasting, @, mean, sum, reshape;
  • partición entrenamiento/prueba y noción de baseline.

Si alguno de estos puntos está débil, usa el Módulo 0 y vuelve a este libro con dos preguntas concretas: qué objeto matemático no entiendes y en qué línea de código aparece.

Resultados de aprendizaje

Al terminar el módulo deberías poder:

  1. Escribir regresión lineal múltiple en forma matricial.
  2. Derivar el gradiente de MSE y la ecuación normal.
  3. Implementar mínimos cuadrados y descenso del gradiente con NumPy.
  4. Diagnosticar curvas de pérdida y problemas de escala.
  5. Explicar condicionamiento y por qué afecta la optimización.
  6. Construir features polinomiales sin mezclar entrenamiento y prueba.
  7. Distinguir underfitting, overfitting, error de entrenamiento y error de validación.
  8. Formular Ridge y Lasso como regularización de un problema de mínimos cuadrados.
  9. Ajustar un modelo logístico para clasificación binaria.
  10. Elegir métricas y umbrales según el costo de falsos positivos y falsos negativos.
  11. Preparar un baseline de proyecto con partición, métrica, modelo y conclusión.

      -> gradiente
        -> descenso del gradiente
          -> escalamiento y condicionamiento
            -> validación
              -> regularización
                -> regresión logística y métricas

Si te bloqueas en un capítulo, busca el nodo anterior en el mapa. Por ejemplo: si Ridge parece opaco, vuelve a mínimos cuadrados; si logística parece opaca, vuelve a pérdida, gradiente y validación.

Reglas de calidad técnica

Antes de entregar un resultado del módulo, verifica:

  • el dataset está separado en entrenamiento y prueba;
  • todo preprocesamiento se ajusta con entrenamiento;
  • existe un baseline simple;
  • reportas al menos una métrica adecuada;
  • la conclusión distingue desempeño numérico e interpretación;
  • el notebook corre desde cero sin depender de variables ocultas;
  • no hay leakage entre entrenamiento, validación y prueba.

Regla de oro. Si no puedes explicar qué minimiza tu algoritmo, qué compara tu métrica y qué decisión habilita tu resultado, todavía no has terminado el análisis.
