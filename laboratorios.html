Laboratorios: trabajar, validar y entregar

Ruta única para los notebooks evaluados de MATE-2105

Entrega oficial

Cada laboratorio se entrega como un único archivo .ipynb en Gradescope, a través del enlace publicado en Bloque Neón. No necesitas saber Git ni crear un repositorio para entregar.

  1. Descarga el notebook desde Materiales.
  2. Completa solo las celdas de respuesta y conserva nombres de funciones.
  3. Reinicia el kernel y ejecuta todo de arriba hacia abajo.
  4. Conserva visibles las salidas del experimento reproducible.
  5. Ejecuta el validador y corrige errores.
  6. Carga el .ipynb en Gradescope y confirma que aparece la recepción.

Gradescope permite reemplazar la entrega antes del cierre cuando la actividad está configurada para reintentos. La nota oficial corresponde a la entrega activa al vencimiento. Un archivo en GitHub, Drive o correo no cuenta como entrega.

Validación

Guarda validar_entrega.py junto al notebook y ejecuta:

python validar_entrega.py lab01_regresion_gradiente.ipynb

El reporte revisa:

  • estructura y laboratorio correcto;
  • una única celda de interpretación;
  • funciones todavía marcadas TODO o NotImplementedError;
  • celdas de respuesta sin ejecutar;
  • salidas de error conservadas;
  • extensión sugerida de la interpretación.

El validador no contiene pruebas ocultas y no calcula la nota. Un aviso puede requerir revisión; un error debe corregirse antes de cargar el archivo.

Qué se califica

El código se evalúa con pruebas deterministas sobre datos distintos de los ejemplos visibles. La interpretación usa una rúbrica corta de tres niveles. Las celdas etiquetadas como experimento producen la evidencia que debes citar; el autograder prueba las funciones por separado para conservar crédito parcial.

Evidencia Método
funciones, formas, valores y estabilidad autograder
respuestas estructuradas de probabilidad y protocolo autograder
interpretación de resultados y límites rúbrica revisada por el profesor

La retroalimentación automática puede señalar una función o propiedad que falla. La parte interpretativa se publica después de la revisión docente.

GitHub opcional

GitHub Classroom sirve para guardar versiones y recibir un check visible en cada push; no es un requisito del curso. La plantilla incluye Codespaces, así que puedes trabajar desde el navegador sin instalar Git localmente.

Descargar plantilla GitHub

Ruta mínima:

  1. acepta el enlace de Classroom;
  2. abre Code > Codespaces > Create codespace;
  3. coloca el notebook en notebooks/;
  4. usa Source Control > Commit & Push;
  5. revisa la acción Validar notebook;
  6. descarga el archivo final y entrégalo en Gradescope.

Si una plataforma falla

Conserva el archivo con salidas visibles y una captura del error de entrega. El profesor anunciará en Bloque Neón una contingencia común para todo el grupo. No envíes versiones distintas por varios canales, porque eso hace ambigua la entrega activa.
