Abrir 20 minutos antes:

Mantener el notebook descargado sin celdas editadas durante la preparación. Si se necesita una copia limpia, volver a descargarla desde el enlace del panel.

Frase de apertura

Hoy quiero que el primer número del modelo aparezca tarde. Antes vamos a predecir su baseline, el tamaño de test y qué información necesitaremos para localizar un error.

Transiciones que conviene decir

  • De distribución a baseline: “La barra más alta ya nos da un competidor sin usar química”.
  • De partición a pausa: “Ahora el problema tiene 45 casos; calculen antes de que Python lo haga”.
  • De tubería a notebook: “Ya sabemos qué debe ocurrir. Abramos el archivo para comprobarlo”.
  • De accuracy a matriz: “Cuarenta y cuatro aciertos dejan un error; queremos saber cuál”.
  • De matriz a límite: “Localizar el error en estas 45 filas no nos dice qué ocurrirá en otro laboratorio”.

Respuestas esperadas

  • La clase mayoritaria es 1, con 71 observaciones.
  • En test hay 45 filas; el baseline acierta las 18 de clase 1.
  • stratify=y conserva aproximadamente proporciones, no garantiza idénticos porcentajes.
  • El escalador aprende media y desviación solo con train porque vive dentro de la tubería.
  • El modelo acierta 44 de 45; el único error es clase 1 predicha como 2.
  • El recall de clase 1 es \(17/18\); la precision de clase 2 es \(12/13\).
  • El resultado no evalúa otros laboratorios, regiones ni periodos.

Errores frecuentes y respuesta docente

Error Respuesta breve
Calcular baseline sobre todo y. Para la intuición inicial sirve; el baseline evaluado debe aprender la moda solo en train.
Escalar antes de separar. Preguntar qué media habría conocido el procedimiento antes de ver test.
Interpretar random_state como mejora estadística. Es reproducibilidad de una partición, no garantía de representatividad.
Leer columnas como observado en la matriz. Volver a los ejes impresos y leer una celda en una oración completa.
Decir que 97.8 % será el desempeño futuro. Pedir tamaño de test, procedencia y cambios no evaluados.
Creer que el notebook “demuestra” regresión logística. Recordar que hoy se usa la implementación; la derivación llega en Semana 6.

Qué recortar si faltan minutos

No recortar la pausa 1, la ejecución de baseline/modelo, la matriz ni el cierre.

Si hay 12 minutos de retraso:

  1. no imprimir el reporte completo; mostrar solo la tabla de slides 22;
  2. resumir slides 25-26 en tres minutos;
  3. dejar la segunda pausa como ticket de salida escrito;
  4. explicar el formato de entrega mediante el anuncio ya publicado.

Contingencias

  • Kernel no inicia: usar las salidas guardadas del notebook y ejecutar solo las aserciones en otro equipo al final. No improvisar cifras.
  • Falla scikit-learn: mostrar python -m pip show scikit-learn, registrar el error y continuar con las figuras congeladas.
  • Muchos estudiantes sin entorno: trabajar en parejas durante el live coding; la entrega individual conserva el reporte de bloqueo.
  • Sin internet: load_wine es local y no requiere descarga.
  • Resultado distinto: detenerse. Revisar semilla, stratify, tamaño de test y versión del notebook antes de interpretar.
