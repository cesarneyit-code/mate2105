Estándar de slides profesionales

Propósito

Esta página define el estándar visual y pedagógico para reescribir las diapositivas del curso. La referencia de calidad usada son los PDFs locales en ejemplos-lectures-mit/, tomados como modelo de arquitectura de clase, no como fuente para copiar contenido o imágenes.

Hallazgos de la referencia

Los ejemplos revisados tienen una lógica consistente:

  1. Formato 16:9 amplio. La página usa aire, márgenes generosos y tipografía grande.
  2. Una idea por slide. Las diapositivas no funcionan como resumen escrito sino como guion visual de explicación.
  3. Progresión por micro-slides. Una clase puede tener 25-60+ slides porque una derivación o diagrama se construye por etapas.
  4. Visual dominante. Cuando hay caso, algoritmo o geometría, la figura ocupa una parte central de la pantalla.
  5. Recap explícito. Las clases abren conectando con el modelo mental anterior.
  6. Diagramas de pensamiento. El estudiante ve cajas, flechas, flujo de datos, flujo de entrenamiento y flujo de decisión.
  7. Fórmulas ancladas. Las ecuaciones aparecen grandes, con poco texto y acompañadas por interpretación.
  8. Cierre operativo. Cada clase termina con qué se espera que el estudiante pueda decir, ejecutar o entregar.

Estándar para MATE-2105

Cada deck debe tener esta arquitectura mínima:

Bloque Función Evidencia esperada
Pregunta de entrada activar el problema aplicado una pregunta grande, no una lista de objetivos
Recap conectar con la clase anterior diagrama o fórmula central anterior
Contrato de la semana aclarar resultados 3 acciones: representar, optimizar, validar, decidir, etc.
Modelo visual dar mapa mental figura, flujo, diagrama o matriz
Notación fijar objetos formas y símbolos verificables
Derivación construir idea matemática pasos separados en micro-slides
Implementación conectar con notebook ruta de funciones o pipeline
Diagnóstico enseñar a leer resultados curva, métrica, matriz o error común
Actividad obligar a razonar ejercicio breve de 5-10 minutos
Cierre sintetizar frase técnica defendible

Reglas visuales

  • Usar fondo blanco y tipografía grande.
  • Evitar slides con más de una idea central.
  • Preferir diagramas grandes sobre listas largas.
  • Usar tarjetas solo cuando separan conceptos comparables; no convertir toda la clase en tarjetas.
  • Mantener fórmulas en bloques grandes y legibles.
  • Incluir al menos una figura, diagrama o flujo fuerte por clase.
  • No usar imágenes de la referencia MIT dentro del curso; solo replicar el estándar pedagógico.
  • Validar con captura visual antes de publicar.

Plan de ejecución

  1. Mantener Módulo 1 como referencia visual y narrativa.
  2. Revisar que cada módulo tenga tema local, figuras propias y rutas portables.
  3. Renderizar todos los decks antes de publicación.
  4. Tomar capturas visuales de portada, figura, fórmula y actividad por módulo.
  5. Sincronizar portal y publicar.

Criterio de aceptación

Un deck se considera profesional cuando:

  • puede dictarse sin leer párrafos largos;
  • tiene secuencia de explicación visible;
  • las fórmulas son legibles desde el fondo del salón;
  • las figuras no se ven como adorno sino como instrumento de razonamiento;
  • cada actividad tiene una decisión clara;
  • el cierre deja una frase técnica que el estudiante puede repetir y defender.
